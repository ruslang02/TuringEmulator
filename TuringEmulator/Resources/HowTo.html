Функции
В программе реализована полноценная система для гибкого редактирования алгоритма и его операций. Присутствуют следующие функции:
– Добавление состояния по нажатии «Добавить...» и «Новое состояние»;
– Изменение состояния путем редактирования значений в ячейках в таблице состояний, где первый столбец – идентификатор состояния, а последующие столбцы – операции при нахождении определенного символа;
– Удаление выделенного состояния при помощи кнопки «Удалить состояние» в меню «Удалить...»;
– Добавление символа в алфавит по нажатии «Добавить...» и «Новый символ»;
– Изменение существующего символа по нажатии правой кнопки мыши по заголовку столбца, вызывающего поле ввода;
– Удаление символа из алфавита при помощи кнопки «Удалить символ» в меню «Удалить...».

Формат ввода
(символ) (операция) (следующая инструкция)
- (символ) - символ, который заменит выбранный символ на ленте
- (операция): < - сдвиг влево; > - сдвиг вправо; . - курсор не меняет положения
- Если вместо этого формата написана команда STOP, то программа завершает исполнение.

Визуализация алгоритма
После каждого изменения алгоритма происходит перерасчёт шагов, при успешном выполнении которого доступна возможность визуализации его работы.
Перед началом визуализации необходимо удостовериться, что программой было выбрано правильное начальное состояние.
Для запуска визуализации необходимо воспользоваться кнопкой «Запуск» (автоматическое исполнение), либо кнопками «Шаг назад» и «Шаг вперед» (ручное исполнение). При автоматическом исполнении имеют место задержки, необходимые для понимания пользователем каждого шага. Время задержки можно изменить при помощи выпадающего меню справа внизу, слева значка «Таймер».
По окончании визуализации программа сообщит об этом, а также изменит значок запуска на «Перезапустить».

Сохранение данных программы
В программе предусмотрена возможность загрузки и сохранения состояния приложения, а также экспорта рассчитанных данных в файл формата JSON. Эти действия можно выполнить через меню «Файл».

При экспортировании в формат JSON сохраняется не только алгоритм и его входные данные, но и также все шаги, которые просчитала программа.
